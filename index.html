Features

Everything you need.
Nothing you don't.

🗺️

World Map Visualization

All your tracks rendered simultaneously on an interactive OpenStreetMap. Pan, zoom, explore — from a city block to a continent. Every color-coded track is spatially indexed in PostGIS for instant queries.

● Leaflet.js OpenStreetMap
📂

Drag & Drop Upload

Drop multiple GPX files at once. Each file gets its own status — ok, skipped, or failed — so you always know what made it in.

💾

Persistent PostGIS Storage

Tracks stored in PostgreSQL with the PostGIS extension. Spatial indexing means queries stay fast even with thousands of routes.

🔑

Privacy by Design Vault

No email. No password. No account. You generate a five-word secret key that lives only in your browser. On the server, only a salted SHA-256 hash is stored — making it impossible to link data back to any individual, even for the operator.

🇪🇺

GDPR Export Workflow

Under GDPR Article 20, your sports watch manufacturer must hand over all your activity data on request. Request your export monthly, drop the GPX files into TrailBlaze, and your adventure map stays up to date — no third-party platform ever needed.

🐳

docker compose up --build — that's it. PostGIS, FastAPI, and the frontend, all wired together automatically.

How it works

From GPX file
to world map in seconds

01
Create your vault
On first visit, click "Create New Private Vault". A cryptographically random five-word key is generated in your browser. Save it — it's the only way to access your data. No email, no password, no account.
02
Request your GDPR export
Under GDPR Article 20 your sports watch manufacturer must provide all your data on request. Download your export — it contains GPX files for every activity you've ever recorded.
03
Upload & visualize
Drag and drop your GPX files onto the upload zone. Each file is parsed, stored in PostGIS under your vault key, and immediately rendered on the map — color-coded by activity type.
04
Explore your trails
Pan across continents, zoom into a single run in Munich, filter by activity. Every adventure you've ever recorded — on one living map that only you can see.

Get started

Up and running in
under two minutes

One command is all it takes. Docker Compose handles the database, the backend, and the frontend automatically.

# Clone the repo
git clone https://github.com/wenzelt/trailblaze-gpx-viewer.git
cd gpx-viewer

# Configure (optional)
cp .env.example .env

# Launch everything
docker compose up --build

# Open in browser
http://localhost:8000
# Upload GPX files
curl -F "files=@track1.gpx"      -F "files=@track2.gpx"      http://localhost:8000/upload

# Fetch all tracks as GeoJSON
curl http://localhost:8000/tracks

# Health check
curl http://localhost:8000/health
# Add to docker-compose.yml under db:
services:
  db:
    image: postgis/postgis
    platform: linux/arm64/v8  # ← add this
    environment:
      ...

# Then rebuild
docker compose up --build
